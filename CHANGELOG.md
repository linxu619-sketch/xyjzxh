# 更新日志 CHANGELOG

本文件记录信阳市建筑装饰装修协会网站的**每一次修改**。
版本号遵循 [语义化版本 SemVer](https://semver.org/lang/zh-CN/)：`主版本.次版本.补丁`
- **补丁 +1**（0.1.0 → 0.1.1）：修 bug、文案/样式微调，不改功能
- **次版本 +1**（0.1.1 → 0.2.0）：新增功能、新页面
- **主版本 +1**（0.x → 1.0.0）：重大改版或不向后兼容的改动

版本号始终与 `package.json` 的 `version` 字段保持一致。

格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/)。

发布节奏：每次有意义的改动**直接新建一个版本号条目并提交**，不使用「未发布」缓冲区。

## [0.6.2] - 2026-05-31

### 变更（迁移到 DeepSeek V4）
- 旧模型 `deepseek-chat` / `deepseek-reasoner` 将于 **2026-07-24 停用**，迁移到 V4：
  - 默认模型改为 `deepseek-v4-flash`（性价比；可配置 `deepseek-v4-pro` 用最高能力）。
  - 思考模式改用请求参数 `thinking:{type:"enabled"|"disabled"}` + `reasoning_effort:"high"`（V4 不再分模型）；流式仍返回 `reasoning_content`，`<think>` 折叠展示逻辑不变。
  - **旧模型名自动映射到 v4-flash**：即使系统设置里仍填 `deepseek-reasoner` 也会自动走 V4（reasoner→开启思考，chat→关闭思考）。
- 已用真实 API 实测通过。

## [0.6.1] - 2026-05-31

### 修复
- **修复 AI 对话偶发报错**：`deepseek.ts` 流在收到 `[DONE]` 后会被重复 `close()`（`return` 又触发 `finally` 再关一次）而抛错，改为 `closed` 标志防止重复关闭。
- 提高 DeepSeek `max_tokens` 1024 → 2048，减少长回答被截断。

### 优化（AI 对话框打磨）
- **思考过程展示**：推理模型的 `reasoning_content` 不再逐 token 加 `🧠` 塞进正文，改为后端包进 `<think>…</think>`、前端渲染成**可折叠的「思考过程」**（思考时显示动画并展开，出答案后自动收起）。
- **报错体验**：对话出错时显示友好的「出错了 · 重试」卡片，一键用上一条消息重试（重构为 `runChat(history)` + `retryLast()`）。
- 正文流式时显示光标；初始加载点保留。
- 确认 AI 对话页为沉浸式（隐藏底栏），用左上角返回键回大厅。

## [0.3.2] - 2026-05-30

### 修复（关键）
- **修复 `启动.bat` 双击启动失败**：根因是该 `.bat` 被存为 UTF-8，中文版 Windows 的 `cmd.exe` 用 GBK 解析时多字节中文导致命令被切碎、报「不是内部或外部命令」、服务器无法启动。改为 **GBK(cp936) 编码 + CRLF**，并去掉无效的 `chcp 65001`。实测双击场景下服务器第 2 秒即就绪。
- 新增 `.gitattributes`：`*.bat -text`，防止 git 改动 `.bat` 字节。
- 在 `CONVENTIONS.md` 记录「`.bat` 必须存 GBK」约定及改后重转命令。
